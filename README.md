# Абитуриент

Веб-приложение для отслеживания позиций абитуриента в списках поступающих медицинских университетов.

## Возможности

- Вход по уникальному ID абитуриента (верификация через Django Admin)
- Синхронизация списков поступающих из медвузов (фоново раз в час + ручное обновление)
- Текущие и прогнозные позиции по направлениям
- Личный кабинет: баллы ЕГЭ, аттестат, выбор вузов с поданными заявлениями

## Стек

- Django 5.2, SQLite, Celery + Redis
- Docker Compose
- Bootstrap + HTMX

## Требования

- Docker и Docker Compose
- Свободный порт **8000**

## Быстрый старт

```bash
# Запуск (сборка + старт всех контейнеров)
make run

# Остановка
make stop

# Перезапуск
make rerun
```

Приложение: http://localhost:8000  
Админка: http://localhost:8000/admin/

## Полезные команды

```bash
make logs              # логи всех сервисов
make build             # пересборка образов
make migrate           # применить миграции
make makemigrations    # создать миграции (все приложения)
make makemigrations app=users   # миграции для одного приложения
make test              # запуск тестов
make shell             # Django shell
make createsuperuser   # создать администратора
```

## Первый запуск

1. `make run`
2. `make createsuperuser` — указать `abiturient_id` и пароль для входа в админку
3. Пользователь входит на сайт по своему ID → админ ставит `is_verified=True`
4. В ЛК указать баллы и вузы с поданными заявлениями
